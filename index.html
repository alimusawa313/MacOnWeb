Your Mac is CI.

MacOn turns the Mac on your desk into a full iOS pipeline — watch a repo, build, test across simulators, ship to TestFlight. Then a playful iPhone & iPad companion lets you monitor it, stream and control its screen, and wake or unlock it from anywhere.

On the Mac

A CI server, without the server.

Everything a hosted CI does for iOS — running on hardware you already own.

Your Mac, your runner

No metered build minutes, no queue, no cloud VM spin-up. Warm caches make it the fastest CI for a single dev.

Bitbucket & GitHub

Watch a branch or open PRs. Poll on a timer, or receive webhooks for instant, push-based builds.

macon.yml pipelines

Bitrise-style workflows: before_run chains, triggers, env, run_if, and always-run steps — defined in your repo.

Test matrix

Fan a step out across device × OS combinations. UI-test on every simulator that matters, in one push.

Ship to TestFlight

fastlane with automatic signing and App Store Connect API keys. Merge to main, upload a build — done.

Script it from any terminal

macon trigger app --follow builds and exits by result. status, logs, cancel, --json, a launchd service, and a Prometheus /metrics endpoint make it a good citizen in bigger setups.

In your pocket

The companion is more than a dashboard.

A free iPhone & iPad app — built like a toy, in a soft 3D clay world you can re-skin — that reaches all the way into the Mac.

Builds, live

Every pipeline and run as a step-by-step timeline with streaming logs. Watch, run, re-run, cancel — even add and edit pipelines with repo & branch pickers.

Stream & control the screen

Hardware-decoded H.264 of the Mac's display, with — on permission — full control: cursor, keyboard, a virtual trackpad, Mac gestures, and a Touch Bar-style strip.

CompactOS — Mac apps, phone-sized

Tap a Mac app and it opens fitted to your device, streaming just that window with big readable text and direct touch. Your Mac's apps, like phone apps.

Shortcuts deck

A Stream Deck-style board of keys that launch apps, fire keyboard shortcuts, or run chains of steps on the Mac — one tap.

Wake & unlock, from anywhere

Keep the Mac reachable, wake it with a Wake-on-LAN packet, and unlock the login screen with a stored password — with an optional privacy curtain the moment it opens.

Always connected, via iCloud

Optional CloudKit sync means a rotated tunnel URL re-points the phone on its own — pair once and it just keeps finding the Mac, no re-typing an address, ever.

Homebrew

Install the CLI.

One tap, one install. Then bootstrap the toolchain and start watching.

1 · Tap & install

# add the tap, then install brew tap alimusawa313/macon https://github.com/alimusawa313/homebrew-macon brew install macon

2 · Check the machine

macon init # installs missing tools macon sims list # iOS / watchOS / tvOS / visionOS

3 · Watch & build

# poll a branch and build every commit macon watch --workspace acme --repo app --branch main # or GitHub, instant, via webhook macon watch --provider github --workspace org \ --repo app --webhook --port 8787

4 · Run it unattended & drive it remotely

# export from the app, run headless as a service macon install-service --config macon-export.json --companion # then, from any terminal macon trigger "My App" --follow # build now, exit 0/1 by result

macon.yml

Pipelines that live in your repo.

Composable workflows and a test matrix — the same file runs in the app or the CLI.

name: PlanPal iOS CI workflows: _setup: steps: - { name: Gems, script: bundle install } - { name: SwiftLint, script: swiftlint lint --strict } test: before_run: [_setup] steps: - name: UI Tests # fan out across devices × OS matrix: device: ["iPhone 17 Pro", "iPad Air 11-inch (M4)"] os: ["26.4", "26.5"] script: bundle exec fastlane test device:"$MACON_MATRIX_DEVICE" os:"$MACON_MATRIX_OS" beta: before_run: [test] steps: - { name: TestFlight, script: bundle exec fastlane beta } triggers: - { pull_request: "*", workflow: test } - { branch: main, workflow: beta }

Yes, opt-in. The Mac can be kept awake so it stays reachable, woken from sleep with a Wake-on-LAN packet, and its login screen unlocked by typing a password you store in the Keychain — after which it can raise a privacy curtain automatically. Turn on iCloud sync and the phone even re-points itself when the tunnel URL rotates, so you never re-type an address. Unlock is best-effort: macOS's Secure Keyboard Entry can block synthetic typing, and it won't defeat a FileVault preboot screen.

It uses a team-only trust model. Every remote power — screen control, CompactOS, wake, unlock — is off by default and gated behind an explicit toggle on the Mac. Secrets and the unlock password live in the macOS Keychain, never in your repo, and your code is only ever checked out on your own machine. A tunnel URL alone grants nothing; pairing codes and device tokens are always required.
